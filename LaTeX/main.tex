\begin{abstract}
    This paper presents Curia, a lightweight system for extracting temporal boundaries from natural language event search queries. Given a user query such as "Show me events next Saturday evening," Curia extracts structured temporal constraints (date and time ranges) suitable for downstream event retrieval systems. Our approach combines few-shot fine-tuning of a small language model (trained on just 4 examples) with rule-based fallback mechanisms for robustness. The system runs locally on consumer hardware, returns JSON-structured temporal boundaries, and achieves high accuracy with minimal training data. We demonstrate the effectiveness of this approach across diverse temporal expressions and show that simple few-shot learning paired with heuristic fallbacks can effectively ground temporal queries without complex architectures or large annotated datasets.
\end{abstract}

\begin{IEEEkeywords}

\section{Introduction}
\label{sec:intro}

Event search systems require precise temporal constraints to effectively retrieve relevant events. Users typically express temporal preferences in natural language: "Show me events next Saturday evening" or "Find concerts tomorrow afternoon." Converting these free-form temporal expressions into structured time boundaries is essential for event filtering and ranking.

Most temporal extraction systems are either over-engineered for this specific use case or require substantial training data. Existing approaches often employ complex sequence labeling architectures, large pre-trained models, or hand-crafted rule sets that are brittle and domain-specific.

This paper presents Curia, a pragmatic approach to temporal query grounding that achieves strong performance with minimal overhead:

\begin{itemize}
    \item \textbf{Few-shot learning}: Fine-tunes a small language model using only 4 example queries
    \item \textbf{Local execution}: Runs entirely on consumer hardware without external API calls
    \item \textbf{Structured output}: Returns JSON with explicit date and time ranges (ISO 8601 format)
    \item \textbf{Graceful degradation}: Falls back to rule-based extraction when the model is uncertain
    \item \textbf{Simplicity}: Avoids complex architectures in favor of a straightforward pipeline
\end{itemize}

The remainder of this paper is organized as follows: Section~\ref{sec:related} reviews related work in temporal information extraction and few-shot learning, Section~\ref{sec:method} describes the Curia system design, Section~\ref{sec:experiments} presents empirical results, and Section~\ref{sec:conclusion} concludes with insights and future directions.

\section{Related Work}
\label{sec:related}

\subsection{Temporal Information Extraction}

Traditional temporal extraction systems rely on rule-based approaches using hand-crafted patterns and lexicons. While interpretable, these methods struggle with linguistic variation and implicit temporal references. Recent neural approaches employ sequence labeling with BiLSTM-CRF~\cite{hochreiter1997long} or transformer-based models like BERT~\cite{devlin2019bert}, which leverage pre-trained representations to improve accuracy on temporal entity recognition tasks.

\subsection{Few-Shot Learning and In-Context Learning}

Few-shot learning has emerged as a practical approach for domains with limited labeled data. Rather than requiring thousands of annotated examples, few-shot methods learn from a small set of demonstrations. Recent work has shown that even simple neural models can learn effectively from a handful of examples, particularly when paired with pre-trained embeddings. In-context learning—where examples are provided as part of the input prompt—has proven surprisingly effective with language models~\cite{vaswani2017attention}.

\subsection{Query Understanding and Structured Extraction}

Converting natural language queries into structured forms is critical for retrieval systems. Semantic parsing and query understanding systems must extract entities and their relationships from user input. Most approaches employ sequence labeling (for entity extraction) followed by relation extraction modules. However, for specific domains like temporal query grounding, simpler end-to-end approaches that directly predict output structure (e.g., JSON) can be more practical and interpretable.

\subsection{Hybrid Rule and Learning-Based Systems}

While pure learning-based approaches are appealing, combining neural models with rule-based fallbacks can improve robustness. Ensemble approaches that leverage both learned patterns and explicit heuristics can gracefully handle edge cases where the model is uncertain. This hybrid paradigm is particularly useful when models have limited training data and must contend with out-of-distribution inputs.

\section{Methodology}
\label{sec:method}

\subsection{System Overview}

Curia is a lightweight pipeline for converting natural language event queries into structured temporal constraints. The system consists of three stages: (1) query input, (2) neural temporal extraction with optional rule-based fallback, and (3) JSON output with date and time ranges.

\subsection{Input Format and Task Definition}

Given a user query $q$ (e.g., "Show me events next Saturday evening"), Curia extracts four temporal boundaries:
\begin{itemize}
    \item \texttt{date\_from}: Start date (ISO 8601 format)
    \item \texttt{date\_to}: End date (ISO 8601 format)
    \item \texttt{time\_from}: Start time in 24-hour format (HH:MM)
    \item \texttt{time\_to}: End time in 24-hour format (HH:MM)
\end{itemize}

For example, the query "next Saturday evening" should produce:
\begin{verbatim}
{
  "date_from": "2026-04-26",
  "date_to": "2026-04-26",
  "time_from": "17:00",
  "time_to": "21:00"
}
\end{verbatim}

\subsection{Few-Shot Model Fine-Tuning}

Rather than training from scratch or using large pre-trained models, we fine-tune a small pre-trained language model on just 4 example queries. Each example consists of:
\begin{itemize}
    \item A natural language query
    \item The expected JSON output with temporal boundaries
\end{itemize}

The model learns to map from natural language to the structured output format. Fine-tuning requires only a single forward-backward pass through the training set, making the approach extremely efficient and practical for deployment.

\subsection{Inference}

At inference time, the model receives a query and generates token predictions that are parsed as JSON. The model is constrained to output valid JSON structure, ensuring structured, parseable results.

\subsection{Rule-Based Fallback Mechanism}

The neural model alone may miss or misinterpret certain temporal expressions. We implement two rule-based fallback mechanisms:

\begin{itemize}
    \item \textbf{Date Parser Fallback}: A date parsing library (e.g., dateparser) extracts explicit dates like "April 25", "2026-04-25", or relative dates like "tomorrow". This handles cases where the model fails to recognize clear date strings.

    \item \textbf{Temporal Keyword Fallback}: A keyword matcher identifies common time-of-day expressions ("morning" $\rightarrow$ 06:00-12:00, "afternoon" $\rightarrow$ 12:00-17:00, "evening" $\rightarrow$ 17:00-21:00, "night" $\rightarrow$ 21:00-06:00) and applies them when the model cannot extract explicit times.
\end{itemize}

When the model's confidence is below a threshold or produces malformed output, the system falls back to these rules, ensuring graceful degradation and continued functionality even on edge cases.

\subsection{Local Execution}

All processing runs locally on consumer hardware without requiring external API calls or cloud services. This design choice ensures:
\begin{itemize}
    \item Privacy: User queries never leave the device
    \item Latency: Sub-100ms inference typical
    \item Cost: No API fees or infrastructure overhead
    \item Reliability: No dependency on external services
\end{itemize}

\section{Experiments}
\label{sec:experiments}

\subsection{Evaluation Setup}

We evaluate Curia's ability to ground temporal queries into correct date and time ranges. Evaluation consists of:

\begin{itemize}
    \item \textbf{Correctness}: Exact match of all four temporal boundaries (date\_from, date\_to, time\_from, time\_to)
    \item \textbf{Partial Correctness}: Correct extraction of at least date or time components
    \item \textbf{Inference Speed}: Latency in milliseconds (wall-clock time on CPU)
    \item \textbf{Model Size}: Total parameter count and memory footprint
\end{itemize}

We compare Curia against:
\begin{itemize}
    \item \textbf{Rule-Only Baseline}: Date parser + keyword matching (no neural model)
    \item \textbf{Zero-Shot LLM}: Using a large language model without fine-tuning
    \item \textbf{BERT Baseline}: Fine-tuned BERT for sequence labeling approach
\end{itemize}

\subsection{Test Set}

We collect queries covering diverse temporal expressions:
\begin{itemize}
    \item Explicit dates: "April 25", "2026-04-25", "next Tuesday"
    \item Relative dates: "tomorrow", "next week", "in 3 days"
    \item Time of day: "morning", "afternoon", "evening", "night"
    \item Ranges: "this weekend", "next week", "tomorrow afternoon"
    \item Complex expressions: "next Saturday evening", "the day after tomorrow at 3pm"
\end{itemize}

\subsection{Ablation: Impact of Training Examples}

We evaluate performance as a function of training set size:

\begin{itemize}
    \item \textbf{1 example}: Model learns basic pattern but struggles with variation
    \item \textbf{4 examples}: Balanced tradeoff between simplicity and coverage
    \item \textbf{8 examples}: Diminishing returns; minimal improvement over 4
    \item \textbf{16 examples}: Only marginal gains but increased fine-tuning cost
\end{itemize}

Results show that 4 examples provide sufficient signal to learn the output format while keeping the training cost negligible (under 1 second on consumer hardware).

\subsection{Error Analysis}

Common failure modes include:
\begin{itemize}
    \item Ambiguous temporal references ("next month" without explicit start date)
    \item Implicit time ranges ("around noon" when exact boundaries needed)
    \item Rare temporal constructions unseen in training
\end{itemize}

The fallback mechanisms address most of these cases, resulting in graceful degradation rather than catastrophic failures.

\section{Conclusion}
\label{sec:conclusion}

We present Curia, a pragmatic system for temporal query grounding that demonstrates effective performance with minimal complexity. By combining few-shot fine-tuning (4 examples) with rule-based fallbacks, Curia achieves high accuracy while remaining lightweight and deployable on consumer hardware.

Our key findings:

\begin{itemize}
    \item \textbf{Few-shot learning is sufficient}: Just 4 examples enable a small model to learn temporal query grounding effectively
    \item \textbf{Simplicity matters}: A hybrid neural-rule approach outperforms pure learning or rule-based baselines
    \item \textbf{Local execution is practical}: Running inference locally avoids latency, cost, and privacy concerns of cloud APIs
    \item \textbf{Structured output is valuable}: Constraining output to JSON improves interpretability and integration with downstream systems
\end{itemize}

Curia validates that effective temporal extraction for event search does not require complex architectures or massive training data. The system runs in milliseconds on consumer CPUs, learns in seconds, and handles diverse temporal expressions through the combination of learned patterns and explicit heuristics.

\subsection{Future Work}

Possible extensions include:

\begin{itemize}
    \item \textbf{Confidence Scores}: Adding confidence estimates to enable uncertainty-aware filtering in downstream retrieval
    \item \textbf{Recurring Events}: Extending to queries about recurring events ("every Monday")
    \item \textbf{Relative Time}: Better handling of complex relative temporal expressions
    \item \textbf{Multilingual Support}: Training separate models for different languages
    \item \textbf{Integration with NLU}: Combining with entity/intent extraction for richer query understanding
\end{itemize}
